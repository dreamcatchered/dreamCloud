- ✅ Worker процесс обрабатывает загрузки в фоне

**Архитектура шифрования**:
1. Файл поступает в `/tmp`
2. Генерируется уникальный ключ файла
3. Файл шифруется локально перед загрузкой
4. Шифрованный файл загружается в Telegram
5. Telegram Message ID сохраняется в БД
6. Оба временных файла (оригинал + зашифрованный) УДАЛЯЮТСЯ

## 🚀 Готово к запуску

### Инструкция запуска:

```bash
# 1. Инициализировать Telegram (если ещё не было)
python init_telegram.py
# Следуй инструкциям для авторизации

# 2. Инициализировать БД
python migrate_db.py

# 3. Запустить приложение
python run.py
```

Приложение запустится на: **http://127.0.0.1:5033**

### Процесс использования:
1. Открыть http://127.0.0.1:5033
2. Кликнуть "Зарегистрироваться"
3. Придумать логин и пароль
4. Войти с учётными данными
5. Загрузить файл кнопкой "Загрузить"
6. Создать папку кнопкой "Папка"
7. Скачать файл через иконку скачивания
8. Поделиться файлом через иконку шаринга

## 🔐 Безопасность

- ✅ **E2E Encryption**: AES-256-GCM на устройстве пользователя
- ✅ **Zero Knowledge**: Сервер никогда не видит расшифрованные данные
- ✅ **Secure Storage**: Файлы хранятся только в Telegram
- ✅ **Password Hashing**: Bcrypt для паролей пользователей
- ✅ **API Keys**: SHA256 хеширование для API ключей
- ✅ **Rate Limiting**: 1000 запросов в час на пользователя
- ✅ **CORS**: Настроен для безопасности

## 📊 API Endpoints

### Аутентификация
```
POST   /api/auth/register              # Регистрация
POST   /api/auth/login                 # Вход
POST   /api/auth/logout                # Выход
GET    /api/auth/me                    # Текущий пользователь
POST   /api/auth/change-password       # Изменить пароль
```

### Файлы
```
GET    /api/files                      # Список файлов и папок
POST   /api/files/upload               # Загрузить файл
GET    /api/files/<id>                 # Информация о файле
GET    /api/files/<id>/download        # Скачать файл
DELETE /api/files/<id>                 # Удалить файл
```

### Папки
```
GET    /api/folders                    # Список папок
POST   /api/folders                    # Создать папку
GET    /api/folders/<id>               # Информация о папке
PUT    /api/folders/<id>               # Обновить папку
DELETE /api/folders/<id>               # Удалить папку
```

### Шаринг
```
GET    /api/shares                     # Мои публичные ссылки
POST   /api/shares                     # Создать публичную ссылку
GET    /api/shares/<id>                # Информация о ссылке
PUT    /api/shares/<id>                # Обновить ссылку
DELETE /api/shares/<id>                # Удалить ссылку
```
